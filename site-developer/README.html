Site Developer

As a site developer, I want to use GitHub to collaborate with my peers on Drupal development.

This chapter walks through:

  • Site setup: Clone an [Acquia Cloud] site repo and getting it set up for local development.
  • Fork: Create a personal "fork" of the site repo on GitHub, for backing up development work and sharing it with others.
  • Pull request: Submit finished work for peer review as a "pull request".
  • Rebase: Update a branch that's behind the the main project, by rebasing.
  • Interactive Rebase: Tidy up a messy commit log before submitting a pull request.
